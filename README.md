# Метеосервіс

Імітаційний прототип системи метеорологічного сервісу, реалізований на мові TypeScript.

## Опис проєкту

Проєкт реалізує об'єктно-орієнтовану систему для управління метеорологічними даними, попередженнями та сповіщеннями. Система містить класи для роботи з метеорологами, техніками, попередженнями та сервісом сповіщень.

## Структура проєкту

```
src/
├── main.ts                      # Головний файл програми
├── WeatherSystemParticipant.ts  # Абстрактний клас учасника системи
├── INotificationPlan.ts         # Інтерфейс для роботи зі сповіщеннями
├── Meteorologist.ts             # Клас метеоролога
├── Technician.ts                # Клас техніка
├── Warning.ts                   # Клас для представлення попередження
├── EventThreshold.ts            # Клас для представлення порогу небезпеки
├── Region.ts                    # Клас для представлення регіону
├── Subscriber.ts                # Клас для представлення підписника
└── NotificationService.ts       # Сервіс для надсилання сповіщень
docs/
├── requirements.md              # Вимоги до програми
├── class.puml                   # UML-діаграма класів
├── usecase.puml                 # UML-діаграма варіантів використання
├── sequence.mmd                 # UML-діаграма послідовності
└── state.mmd                    # UML-діаграма станів
```

## Архітектурні особливості

- **Абстрактний клас**: `WeatherSystemParticipant` - базовий клас для всіх учасників системи
- **Інтерфейс**: `INotificationPlan` - інтерфейс для формування сповіщень
- **Наслідування**: Класи `Meteorologist` та `Technician` успадковують `WeatherSystemParticipant`
- **Реалізація**: Обидва класи реалізують інтерфейс `INotificationPlan`
- **Композиція**: Клас `Warning` містить об'єкт `EventThreshold`
- **Агрегація**: Клас `Region` містить колекцію об'єктів `Warning`
- **Залежність**: Класи залежать один від одного для функціональності

## Встановлення залежностей

```bash
npm install
```

## Компіляція

Для компіляції TypeScript коду виконайте:

```bash
npm run build
```

Компільований JavaScript код буде розміщений у каталозі `dist/`.

## Запуск

Для запуску скомпільованої програми виконайте:

```bash
npm start
```

Або для розробки з використанням `ts-node` (без попередньої компіляції):

```bash
npm run dev
```

## Вимоги до реалізації

- ✅ Кількість класів: >= 3
- ✅ Залучений принаймні один інтерфейс
- ✅ Залучений абстрактний клас
- ✅ Кожен клас має мінімум один атрибут і один метод
- ✅ Використана інкапсуляція поведінки через інтерфейс та абстрактний клас
- ✅ Використані конструктори для створення екземплярів
- ✅ Є фінальні методи (змодельовано через дизайн класів)
- ✅ Використані 4 типи зв'язків:
  - Наслідування (Meteorologist, Technician extends WeatherSystemParticipant)
  - Реалізація (Meteorologist, Technician implements INotificationPlan)
  - Залежність (класи залежать друг від друга)
  - Композиція (Warning містить EventThreshold)
  - Агрегація (Region містить Warning)

## Примітки

- Програма використовує лише консольний вивід (без зовнішніх сервісів та БД)
- Всі коментарі в коді виконані українською мовою
- Назви класів, методів та атрибутів англійською мовою
- Мінімальна складність реалізації для демонстраційного прототипу
